Clon Trello

Este proyecto lo he hecho en su totalidad con la IA Gemini. A continuación mostraré los prompt´s que le fui enviando a Gemini para crear este Visor de documentos además de todos los problemas que me fui encontrando a lo largo de este camino.

Este proyecto tiene una sola versión, la he llamado MK-AGG (por Antigravity y por la IA que está incorporada, Gemini). ¿Por qué solo usé Gemini teniendo también Claude 4.5? Porque comencé con un chat normal de Gemini para pedirle ideas y una vez me dio estas ideas y un primer código para trabajar me pasé a Antigravity. Sin embargo, nunca abandoné del todo el primer chat de Gemini ya que seguí haciéndole consultas sobre qué APIs podría introducir al proyecto y para que me siguiese dando ideas.

Estas son alguna ideas que me dio:

Editor de Documentos "Nativo" (Alternativa a Google Docs)

  • La Solicitud: Querías integrar la API de Google Docs para editar documentos.
  • El Problema: Google prohíbe incrustar su editor dentro de otras webs (no se puede usar iframe).
  • La Solución Propuesta (Implementada): Usar Quill.js + Firebase Firestore.
    • Permite tener un editor de texto rico (negritas, listas, títulos) dentro de tu aplicación sin que el usuario salga.
    • Implementamos autoguardado ("sinapsis activa") que guarda los cambios en la nube automáticamente al dejar de escribir.

Ideas de Accesibilidad (Neuro-inclusividad)

Adaptamos las ideas para encajar con tu diseño limpio y moderno (tipo SaaS/Notion):

  • Modo Dislexia: Un botón que cambia la tipografía a OpenDyslexic y aumenta el espaciado entre líneas y letras para facilitar la lectura.
  • Modo Sepia (Descanso Visual): Cambiar el fondo blanco brillante (#ffffff) a tonos crema/café suaves para evitar fatiga visual y migrañas.
  • Guía de Lectura: Una barra horizontal semitransparente (estilo resaltador amarillo) que sigue al ratón para ayudar a no perder la línea al leer tablas o textos largos.
  • Sintetizador de Voz (Text-to-Speech): Un botón en el editor que lee el contenido del documento en voz alta.

Análisis de Integración Google Drive / Picker API

  • La Idea: Añadir botones para "Seleccionar desde Drive" o editar archivos de Drive.
  • El Veredicto: Se recomendó NO hacerlo en esta etapa por tres razones:
    1. Experiencia de Usuario: Obliga al usuario a salir de tu web para editar (se abre una pestaña nueva de Google).
    2. Seguridad: Google exige una auditoría de seguridad costosa y compleja para apps que modifican archivos de Drive.
    3. Pérdida de Control: Es difícil sincronizar tus permisos de carpetas "Cortex" con los permisos reales de Google Drive.

Al final las APIs que se usaron fueron Google Drive API y Google Picker API

Primer diseño:

El primer chat de Gemini me dio este boceto, y me gustó, por lo que fui trabajando sobre él hasta construir el Visor final. La idea que se me ocurrió fue hacerlo colaborativo, pero antes de llegar a eso, le fui pidiendo otras cosas a Antigravity:

Prompts:

Prompts primera  parte

  • Depurar Pantalla Blanca:
    Identificar y arreglar el error de sintaxis que impedía cargar la lista de carpetas.
  • Corregir ReferenceError:
    Solucionar el error "allFoldersCac he is not defined"
  •  
  • Manejo de Errores de Permisos:
    Evitar el fallo al cargar participantes, sin permisos, mostrando "usuario protegido".
  • Añadir Opción Descargar:
    Incorporar el botón de descargar a los archivos.
  •  
  • Mejorar Estética Menú Hamburguesa::
    Estabilizar el botón de menú de carpeta.
  • Arreglar Superposición de Menús
    Corregir el Z index para que los menús no se corten

Prompts última parte

  • Igualar estilos visuales:
    Pediste que el efecto hover (fondo al pasar el ratón) en las opciones de carpeta fuera del mismo color que el de los archivos.
  • Arreglar integración con Google Drive:
    Trabajamos en resolver los errores de permisos y descarga, ajustando el scope de la API y el ID de cliente.
  •  
  • Implementar "Renombrar Archivo":
    Solicitaste añadir la opción para cambiar el nombre a los documentos.
  •  
  • Corregir botón "Renombrar":
    Me avisaste de que el botón no hacía nada (faltaba el evento), y lo arreglé.
  • Añadir soporte para SQL y Java:
    Pediste que la aplicación reconociera y permitiera trabajar con estos formatos.
  • Arreglar selección local:
    Me indicaste que no podías seleccionar archivos .sql, .java ni .xsl desde tu ordenador, y actualicé los filtros.
  • Crear "Visor de Código":
    Solicitaste poder "ver" el contenido de los archivos de código directamente en la app.
  • Solucionar error de carga (CORS):
    Reportaste el error Failed to fetch al abrir el visor. Te guié para configurar los permisos del bucket de Firebase mediante la consola.
  • Añadir botón "Copiar":
    Pediste un botón para copiar todo el código al portapapeles desde el visor.
